## 2. 实验原理

### 2.1 寄存器 (Register)

FPGA 上配置了大量的触发器资源来实现数据的存储。
* **基本单元**：D 触发器可以用于存储比特信号。
* **寄存器构建**：给 D 触发器加上置数功能就变成了一位寄存器。

#### 门控反馈工作原理
根据置数（`load`）信号的状态，寄存器内部的硬件逻辑行为如下：
* 当 `load` 信号为 `1` 时，输入信号 `in` 被送入或门中。 由于或门的另一个输入端为 `0`，此时触发器输入端满足 `D = in`，所以在下一个时钟周期里，输出端状态刷新为 `q = in`。
* 当 `load` 值为 `0` 时，当前输出值 `q` 被反馈到或门中。 由于或门的另一个输入值为 `0`，此时触发器输入端满足 `D = q`，因此在下一个时钟周期里，`q` 值保持先前的值不变。

### 2.2 存储器 (Memory)

存储器是一组存储单元的集合，用于在计算机中存储二进制的数据。

#### 接口端口构成
* **输入端口**：包括读/写地址端口、数据输入端口等。
* **输出端口**：一般指的是数据输出端口。
* **控制端口**：包括时钟端（`clk`）和读/写控制端口。

#### 核心工作过程
* **写数据**：在时钟（`clk`）的有效沿（上升或下降沿），如果写使能（`Wr_en`，部分设计也可以没有使能端）有效，则读取输入总线（`Data_in`）上的数据，将其存储到输入地址线（`In_addr`）所指的存储单元中。
* **读数据**：存储器的输出可以受时钟和使能端的控制，也可以不受其控制。
  * *受时钟控制*：在时钟有效沿，将输出地址所指示的单元中的数据输出到输出总线（`Data_out`）上。
  * *不受时钟控制*：只要输出地址有效，就立即将此地址所指单元中的数据送到输出总线（`Data_out`）上。

#### 常见工作模式
FPGA 存储器支持多种配置模式以适应不同的应用场景：
* 真双口 RAM (True Dual-Port RAM) 
* 简单双口 RAM (Simple Dual-Port RAM) 
* 单口 RAM (Single-Port RAM) 
* ROM (只读存储器) 
* FIFO 缓存 

### 2.3 核心差异：寄存器组 vs 主存

| 特性维度 | 寄存器 / 寄存器组 | 主存储器 |
| :--- | :--- | :--- |
| **容量大小** | 通常容量较小。 | 容量一般较大。 |
| **存取速度与时延** | 要求存取速度快，存取时延必须控制在 1 个时钟周期内。 | 读写时间较长。 |
| **并行访问要求** | 并行访问要求高。 例如单周期 CPU 每个时钟周期往往要求同时读取 2 个通用寄存器并完成 1 个寄存器的写回。 | 读写过程遵循严格的时序要求。 |
| **特殊时序输出** | 在要求较高时，结果可能需要异步输出，即不随时钟沿读取，输出随输入地址实时改变。 | 无法大规模支持异步复杂输出，必须满足结构化时序。 |
| **资源消耗率** | 在这样高的要求下，寄存器组的大小不可能太大，否则会消耗非常多的资源。 | 针对大容量进行了结构优化，单位容量的资源消耗率较低。 |
